# Audit Logs

## Objectif

Toutes les actions importantes doivent être enregistrées.

---

## Actions enregistrées

- Connexion
- Déconnexion
- Création
- Modification
- Suppression
- Impression
- Exportation
- Validation
- Paiement
- Annulation

---

## Informations enregistrées

Chaque journal contient :

- Utilisateur
- Tenant
- Module
- Action
- Ancienne valeur
- Nouvelle valeur
- Adresse IP
- Navigateur
- Date
- Heure

---

## Exemple

Utilisateur :

Jean Pierre

Action :

Suppression d'un client

Date :

15 juillet 2026

Heure :

14:35

IP :

192.168.1.10

---

## Sécurité

Les Audit Logs ne peuvent jamais être modifiés.

Ils peuvent uniquement être consultés.

---

## Règle

Toute opération importante doit produire automatiquement un Audit Log.
